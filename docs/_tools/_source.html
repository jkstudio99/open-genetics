# การติดตั้ง

ความต้องการของระบบ

Requirement Version Notes
PHP >= 8.1 pdo, json, mbstring
MySQL >= 5.7 หรือ MariaDB 10.3+
Composer >= 2.0 Dependency manager
Apache XAMPP / any mod_rewrite enabled

เริ่มต้นใน 4 คำสั่ง

วิธีแนะนำ — composer create-project สร้างโปรเจกต์พร้อมใช้งานทันที ไม่ต้องตั้งค่า PATH

1. สร้างโปรเจกต์

composer create-project open-genetics/framework my-api
cd my-api

2. ตั้งค่า Environment

สร้างฐานข้อมูลใน phpMyAdmin แล้วแก้ไขค่า DB_NAME, DB_USER, DB_PASS และ JWT_SECRET

cp .env.example .env

3. Bootstrap ระบบ

สร้างตาราง + Seed RBAC roles + Admin user อัตโนมัติ — ใช้เวลา <2 วินาที

php genetics mutate

4. เริ่ม Dev Server

สร้างไฟล์ใน api/ แล้วได้ endpoint ทันที — ไม่ต้องประกาศ route

php genetics serve
# → Dev server รันที่ http://127.0.0.1:8080
# → สร้าง api/hello.php แล้ว GET /api/hello พร้อมใช้ทันที

Default Admin: admin@opengenetics.io / password — เปลี่ยนรหัสผ่านทันทีหลัง deploy ด้วย php genetics make:user

Global Installer (ตัวเลือกสำหรับ Power Users)

ถ้าสร้างโปรเจกต์ OpenGenetics บ่อยๆ ติดตั้ง genetics CLI แบบ global เพื่อใช้คำสั่ง genetics new จากทุกที่

# ติดตั้ง CLI แบบ global (ต้องเพิ่ม ~/.composer/vendor/bin ใน PATH)
composer global require open-genetics/framework

# สร้างโปรเจกต์ใหม่จากทุกที่
genetics new my-project
cd my-project

แนะนำ: ใช้ composer create-project สำหรับผู้เริ่มต้นและ XAMPP environment ใช้ composer global require เมื่อต้องสร้างโปรเจกต์ใหม่บ่อยและตั้งค่า PATH ไว้แล้ว

# Architecture

OpenGenetics ใช้ File-based Routing — URL path ตรงกับ file path ใน api/ โดยตรง ไม่มี route registry

HTTP Method + URL File Class Method
POST /api/auth/login api/auth/login.php post(array $body)
GET /api/users api/users.php get(array $query)
PUT /api/users api/users.php put(array $body)
DELETE /api/users api/users.php delete(array $body)

Endpoint Template

use OpenGenetics\Auth\Guard;
use OpenGenetics\Core\{Database, Response};

class MyEndpoint {
    public static function get(array $query): void
    {
        Guard::requireAuth();
        $data = Database::query("SELECT * FROM my_table");
        Response::success($data);
    }

    public static function post(array $body): void
    {
        Guard::requireRole(Guard::ADMIN);
        Response::success(null, 'Created', 201);
    }
}

# Middleware Pipeline

ระบบ Middleware แบบ Chain of Responsibility — ทุก request ผ่าน pipeline ก่อนถึง endpoint โดย CORS เป็น global middleware ที่รันทุก request, Auth/RateLimit ประกาศผ่าน #[Middleware] attribute รองรับ #[SkipMiddleware] และ Pipeline::after()

Request Lifecycle

HTTP Request
    ↓
Apache .htaccess  →  public/index.php
    ↓
Pipeline::addGlobal(CorsMiddleware::class)   ← Global middleware
    ↓
Router::dispatch()  →  resolve api/{path}.php
    ↓
Collect #[Middleware] attributes from endpoint class
    ↓
Pipeline::send($body)
  →  CorsMiddleware::handle()     ← Sets CORS headers
  →  AuthMiddleware::handle()     ← Validates JWT, sets Guard user
  →  RateLimitMiddleware::handle() ← Checks file-based rate limit
    ↓
Class::method($body)               ← Endpoint handler
    ↓
Response::success() / error()
    ↓
JSON Output

การใช้งาน #[Middleware] Attribute

use OpenGenetics\Core\Middleware;

// ต้อง login เท่านั้น
#[Middleware('auth')]
class Products
{
    public static function get(array $body): void
    {
        Response::success(['data' => '...']);
    }
}

// เฉพาะ Admin เท่านั้น
#[Middleware('auth:admin')]
class AdminPanel { ... }

// Auth + Rate Limit 10 req/60s
#[Middleware('auth', 'rate:10,60')]
class SensitiveEndpoint { ... }

Middleware ที่มีให้ใช้งาน

Alias Class Params การใช้งาน
cors CorsMiddleware — CORS headers + preflight (global)
auth AuthMiddleware role1,role2 ตรวจ JWT + role อาจระบุหรือไม่ก็ได้
rate RateLimitMiddleware max,window จำกัด request ต่อ IP (default 60/60s)

สร้าง Middleware ใหม่

# Scaffold ด้วย CLI
php genetics make:middleware Cache
# → สร้าง src/Middleware/CacheMiddleware.php
final class CacheMiddleware
{
    public function handle(array $request, callable $next): void
    {
        // ตรวจ cache ก่อน
        $cached = Cache::get($cacheKey);
        if ($cached) {
            Response::success($cached);
            return;
        }
        $next($request);
    }
}

// Register alias ใน index.php
Pipeline::alias('cache', CacheMiddleware::class);

Middleware ทุกตัวต้องมี handle(array $request, callable $next) — เรียก $next($request) เพื่อส่งต่อ pipeline หรือไม่เรียกเพื่อ short-circuit

#[SkipMiddleware] — ยกเว้น Global Middleware

ใช้ exclude middleware ที่ register ไว้แบบ global ออกจาก endpoint ใดเป็นการเฉพาะ

use OpenGenetics\Core\SkipMiddleware;
use OpenGenetics\Middleware\CorsMiddleware;

// Webhook endpoint ที่ไม่ต้องการ CORS headers
#[SkipMiddleware(CorsMiddleware::class)]
class WebhookStripe
{
    public static function post(array $body): void
    {
        // process Stripe webhook...
    }
}

LogMiddleware — บันทึก Access Log

เขียน access log ไปที่ storage/logs/access-YYYY-MM-DD.log ทุก request รองรับทั้ง global และ per-endpoint

// ใน public/index.php — log ทุก request
Pipeline::alias('log', LogMiddleware::class);
Pipeline::addGlobal(LogMiddleware::class);

// หรือเฉพาะ endpoint ที่ต้องการ
#[Middleware('log')]
class AdminUsers { ... }
# ตัวอย่าง access log: storage/logs/access-2026-02-24.log
[2026-02-24 14:23:01] 127.0.0.1 GET /api/users → 200 (12.4ms)
[2026-02-24 14:23:05] 127.0.0.1 POST /api/auth/login → 401 (3.1ms)

Pipeline::after() — Post-Response Hooks

Register callbacks ที่รันหลัง response ส่งออก — เหมาะสำหรับ audit log, metrics, cleanup โดยไม่ block response

// ใน public/index.php
Pipeline::after(function (?\Throwable $error): void {
    // ถ้า $error !== null แสดงว่า request ล้มเหลว
    AuditLog::record($_SERVER['REQUEST_URI'], $error?->getMessage());
});

ErrorHandler::reporter() — Custom Reporter

ต่อ error ไปยัง Sentry, Bugsnag, Slack หรือ service ใดก็ได้ โดยไม่กระทบ HTTP response

// ใน public/index.php
ErrorHandler::reporter(function (\Throwable $e): void {
    // Send to Sentry, Slack, etc.
    Sentry::captureException($e);
});

// Log file: storage/logs/app-2026-02-24.log (auto-created)
// [2026-02-24 14:23:05] ERROR RuntimeException: User not found in .../api/users/{id}.php:22

# Database

เข้าถึงฐานข้อมูลได้ 2 วิธี: Query Builder (DB::table()) สำหรับ dynamic queries ที่อ่านง่าย และ Raw SQL (Database::query()) สำหรับ queries ที่ซับซ้อน — ทั้งคู่ใช้ PDO prepared statements ป้องกัน SQL Injection โดยค่าเริ่มต้น

query() — Fetch All

$users = Database::query(
    "SELECT * FROM users WHERE role_id = :role AND is_active = :a",
    ['role' => 2, 'a' => 1]
);

queryOne() — Fetch Single Row

$user = Database::queryOne(
    "SELECT * FROM users WHERE id = :id",
    ['id' => 1]
);

execute() — Insert / Update / Delete

Database::execute(
    "UPDATE users SET is_active = :a WHERE id = :id",
    ['a' => 0, 'id' => 5]
);

transaction()

Database::transaction(function (PDO $pdo) {
    Database::execute("UPDATE accounts SET balance = balance - 100 WHERE id = 1");
    Database::execute("UPDATE accounts SET balance = balance + 100 WHERE id = 2");
});

LIMIT/OFFSET: ใช้ intval() guard เพราะ MySQL ไม่รองรับ named params ใน LIMIT เมื่อ EMULATE_PREPARES=false

# Query Builder NEW v2.3

OpenGenetics Query Builder — API ที่ออกแบบมาเพื่อ Developer Experience โดยเฉพาะ ผสมแนวคิดจาก FluentPDO (auto-join shortcuts), CodeIgniter (operator-in-key, when()), และ Pixie (find(), findAll()) แต่มีเอกลักษณ์ของ OpenGenetics เอง

3 ระดับการใช้งาน

Static One-Liners

DB::find(), DB::insert() — จบบรรทัดเดียว

Fluent Chain

where(), when(), search(), sort() — สำหรับ 80% use cases

Advanced Join

leftJoin() 3-arg, toSql() debug — สำหรับ query ซับซ้อน

where() — Array-based + Operator-in-Key

ส่ง conditions เป็น array ทีเดียว — รองรับ operator ใน key, auto IN(), auto IS NULL

// Array-based: ส่งหลาย conditions ทีเดียว
DB::table('tickets')->where([
    'status'        => 'open',                        // = (default)
    'priority'      => ['high', 'critical'],           // auto IN()
    'created_at >=' => '2025-01-01',                  // operator-in-key
    'deleted_at'    => null,                           // auto IS NULL
    'email !'       => null,                           // IS NOT NULL
])->get();

// Operator-in-key แบบ string (ทีละ condition)
DB::table('users')
    ->where('age >=', 18)         // WHERE age >= 18
    ->where('name !=', 'test')     // WHERE name != 'test'
    ->get();

// Backward compatible — เดิมยังใช้ได้ 100%
DB::table('users')->where('status', 'active')->get();
DB::table('users')->where('age', '>=', 18)->get();

when() — Conditional Filter (ไม่ต้อง closure)

Apply filter เมื่อ $condition เป็น truthy เท่านั้น — ข้าม null, '', 0, false อัตโนมัติ

DB::table('tickets')
    ->when($status,     ['status'   => $status])    // ถ้า $status มีค่า → apply
    ->when($priority,   ['priority' => $priority])  // ถ้า null → ข้ามไป
    ->when($dateFrom,   ['created_at >=' => $dateFrom])
    ->get();

// ใช้ closure ก็ได้ถ้าต้องการ logic ซับซ้อน
->when($search, fn($q) => $q->search(['name', 'email'], $search))

search() — Multi-column LIKE (เอกสิทธิ์ OpenGenetics)

ค้นหาข้ามหลาย column ด้วยคำสั่งเดียว — ข้ามอัตโนมัติเมื่อ $keyword ว่าง

DB::table('tickets')
    ->search(['subject', 'ticket_number', 'description'], $keyword)
    // → WHERE (subject LIKE '%keyword%' OR ticket_number LIKE '%keyword%' OR description LIKE '%keyword%')
    ->get();

sort() — Prefix Notation (เอกสิทธิ์ OpenGenetics)

DB::table('tickets')
    ->sort('-created_at')              // ORDER BY created_at DESC
    ->sort('name')                     // ORDER BY name ASC
    ->sort('-priority', '-created_at') // ORDER BY priority DESC, created_at DESC

leftJoin() — 3-arg Shortcut

// ใหม่: 3 args — ลด = ออก (เพราะ join ใช้ = เสมอ)
DB::table('tickets AS t')
    ->leftJoin('categories AS c', 'c.id', 't.category_id')
    ->leftJoin('users AS u', 'u.id', 't.reporter_id')
    ->get();

// เดิม 4 args ยังใช้ได้
->leftJoin('categories', 'tickets.category_id', '=', 'categories.id')

find() / findAll() — Instance Shortcuts

// find by primary key → single row or null
$user = DB::table('users')->find(3);

// findAll by column → array of rows
$admins = DB::table('users')->findAll('role_name', 'ADMIN');

Static One-Liners — DB class

// READ
$user  = DB::find('users', 3);                          // SELECT * WHERE id = 3
$user  = DB::find('users', 3, ['id', 'email']);          // SELECT id, email WHERE id = 3
$users = DB::findAll('users', ['is_active' => 1]);        // SELECT * WHERE is_active = 1
$email = DB::value('users', 'email', ['id' => 3]);         // single column value
$total = DB::count('users', ['is_active' => 1]);          // COUNT
$has   = DB::exists('users', ['email' => 'j@test.com']);  // bool

// WRITE
$id    = DB::insert('users', ['name' => 'Jo', 'email' => 'j@test.com']);
DB::update('users', ['name' => 'New'], ['id' => 3]);
DB::delete('users', ['id' => 3]);
DB::softDelete('users', ['id' => 3]);

toSql() / getBindings() — Debug

$q = DB::table('tickets')
    ->where(['status' => 'open', 'priority >=' => 2])
    ->sort('-created_at');

var_dump($q->toSql());        // SELECT * FROM `tickets` WHERE `status` = ? AND `priority` >= ? ORDER BY `created_at` DESC
var_dump($q->getBindings());  // ['open', 2]

Before / After: Tickets Endpoint

// ❌ BEFORE — Raw SQL ~40 บรรทัด
$where = ['t.deleted_at IS NULL'];
if (!empty($_GET['status']))   { $where[] = 't.status = ?'; $params[] = $_GET['status']; }
if (!empty($_GET['priority'])) { $where[] = 't.priority = ?'; $params[] = $_GET['priority']; }
if (!empty($_GET['search']))   { $where[] = '(t.subject LIKE ? OR t.ticket_number LIKE ?)'; ... }
// ... 30+ more lines

// ✅ AFTER — OpenGenetics Query Builder ~15 บรรทัด
$result = DB::table('tickets AS t')
    ->select('t.*', 'c.name AS category_name', 'u.email AS reporter_email')
    ->leftJoin('categories AS c', 'c.id', 't.category_id')
    ->leftJoin('users AS u', 'u.id', 't.reporter_id')
    ->where(['t.deleted_at' => null])
    ->when($user['role_name'] === 'EMPLOYEE', ['t.reporter_id' => $user['id']])
    ->when($status,     ['t.status'   => $status])
    ->when($priority,   ['t.priority' => $priority])
    ->when($dateFrom,   ['t.created_at >=' => $dateFrom])
    ->search(['t.subject', 't.ticket_number'], $search)
    ->sort($sortExpr)
    ->paginate($perPage, $page);

Zero Breaking Changes: ทุก method เดิมยังใช้ได้ 100% — where('col', 'val'), leftJoin(4 args), select(array), orderBy() ฯลฯ ไม่มีอะไรแตก

Classic API (v2.2)

Fluent, chainable query API — เขียน query น้อยลง อ่านง่ายขึ้น โดยยังคง enforce prepared statements ทุก query เพื่อป้องกัน SQL Injection

Fetch — get() / first() / count()

use OpenGenetics\Core\DB;

// Fetch all rows matching the criteria
$users = DB::table('users')
    ->select(['id', 'email', 'role_name'])
    ->where('is_active', 1)
    ->where('tenant_id', $tenantId)
    ->orderBy('created_at', 'DESC')
    ->get();

// First matching row — returns ?array (null if not found)
$user = DB::table('users')->where('email', $email)->first();

// Count rows without fetching them
$total = DB::table('users')->where('is_active', 1)->count();

paginate() — Zero-Config Pagination

// Reads ?page= from $_GET automatically. Returns data + meta in one call.
$result = DB::table('users')
    ->where('is_active', 1)
    ->orderBy('created_at', 'DESC')
    ->paginate(20);  // 20 items per page

// Send the response — meta is already populated
Response::paginatedFrom($result);

WHERE Variants

DB::table('products')
    ->where('price', '>=', 100)                    // operator form
    ->whereIn('status', ['active', 'pending'])   // IN (...)
    ->whereLike('name', '%phone%')               // LIKE
    ->whereNotNull('published_at')               // IS NOT NULL
    ->where('deleted_at', null)                  // IS NULL
    ->get();

Insert / Update / Delete

// Insert — returns the new auto-increment ID
$id = DB::table('users')->insert([
    'email'     => 'alice@example.com',
    'role_name' => 'EMPLOYEE',
]);

// Update — returns affected row count
DB::table('users')
    ->where('id', $id)
    ->update(['role_name' => 'MANAGER']);

// Hard delete matching rows
DB::table('sessions')->where('user_id', $id)->delete();

Soft Delete

// Set deleted_at = NOW()
DB::table('users')->where('id', $id)->softDelete();

// Clear deleted_at (restore)
DB::table('users')->where('id', $id)->restore();

// Scope out soft-deleted rows
DB::table('users')->withoutTrashed()->get();

// Static helper shortcut
Database::softDelete('users', $id);

ยังคง type-safe ครบ: Query Builder ใช้ Database::query() ด้านใน — ทุก value ผ่าน ? placeholder เสมอ ไม่มีความเสี่ยง SQL Injection แม้จะเขียน query แบบ fluent

# Guard / RBAC

Guard ตรวจสอบ JWT token และ role ก่อนเข้า endpoint — ใช้ requireAuth() / requireRole() แบบ strict (throw 401/403) หรือ Guard::check() แบบ soft (คืนค่า bool) สำหรับ endpoint ที่ auth เป็น optional 

use OpenGenetics\Auth\Guard;

// ต้อง login เท่านั้น (role ใดก็ได้)
Guard::requireAuth();

// ต้องเป็น Admin เท่านั้น
Guard::requireRole(Guard::ADMIN);   // role_id = 1

// ต้องเป็น Manager หรือสูงกว่า
Guard::requireRole(Guard::MANAGER);      // role_id = 2

// ดึงข้อมูล user ปัจจุบัน
$user = Guard::user();
// ['id'=>1, 'email'=>'...', 'role_id'=>1, 'role_name'=>'admin']

// ตรวจสอบ role โดยไม่ throw
if (Guard::hasRole(Guard::ADMIN)) {
    // admin-only logic
}

// check() — ตรวจสอบว่า login อยู่หรือไม่ โดยไม่ throw (v2.2+)
if (Guard::check()) {
    $user = Guard::user();    // safe ใช้ใน optional-auth endpoints
} else {
    // ไม่ได้ login — แสดง public data
}
Method Description
Guard::requireAuth() บังคับ login — throw 401 ถ้าไม่มี token
Guard::requireRole($role) บังคับ role — throw 403 ถ้าสิทธิ์ไม่เพียงพอ
Guard::check() คืนค่า bool — ไม่ throw exception เหมาะสำหรับ optional auth
Guard::user() ดึง payload ของ user จาก JWT
Guard::hasRole($role) ตรวจสอบ role โดยไม่ throw — คืนค่า bool
Guard::id() ดึง user ID จาก JWT

Role Constants

Constant role_id Description
Guard::ADMIN 1 ผู้ดูแลระบบ — สิทธิ์สูงสุด
Guard::MANAGER 2 Manager — ผู้จัดการแผนก
Guard::EMPLOYEE 3 พนักงานทั่วไป

# Caching Layer

File-based cache พร้อม TTL, tag-based invalidation, remember() pattern และ Cache::namespace() สำหรับแยก key space ระหว่าง module — ไม่ต้องติดตั้ง Redis หรือ Memcached

การใช้งานพื้นฐาน

// เก็บ cache 5 นาที
Cache::put('user:1', $userData, 300);

// ดึง cache
$user = Cache::get('user:1');

// remember() — ดึงจาก cache หรือ compute ถ้าหมดอายุ
$products = Cache::remember('products:all', 300, function() {
    return Database::query("SELECT * FROM products");
});

// Tag-based flush
Cache::tag('products')->put('list', $data, 300);
Cache::flush('products'); // ลบทุก entry ที่ tag=products

Namespace Prefix

// namespace() — เพิ่ม prefix เพื่อแยก cache key ระหว่าง modules
Cache::namespace('users')->put('list', $data, 300);   // key = "users:list"
Cache::namespace('users')->get('list');               // ดึง key "users:list"
Cache::namespace('users')->flush();                    // flush ทุก key ใน namespace

// เทียบเท่ากับ tag() แต่อ่านง่ายกว่าสำหรับ module isolation
Cache::namespace('products')->remember('featured', 600, function() {
    return DB::table('products')->where('featured', 1)->get();
});

API Reference

Method การใช้งาน
Cache::put(key, val, ttl) เก็บ cache พร้อม TTL (วินาที, 0=forever)
Cache::get(key, default) ดึง cache หรือ default ถ้าหมดอายุ
Cache::remember(key, ttl, fn) ดึง/คำนวณ+เก็บ ถ้าไม่พบ
Cache::namespace(prefix) สร้าง namespaced cache instance เพื่อป้องกัน key collision
Cache::forget(key) ลบ cache entry เดียว
Cache::flush(tag) ลบทุก entry ของ tag
Cache::clear() ล้าง cache ทั้งหมด
Cache::gc() ลบ expired entries
Cache::stats() แสดงสถิติ cache

CLI Commands

php genetics cache:clear    # ล้าง cache ทั้งหมด
php genetics cache:stats    # แสดง disk size, entries, tags
php genetics cache:gc       # ลบ expired entries

# Field Selector (GraphQL-lite)

Sparse Fieldsets ผ่าน ?fields= query param — ลด response payload โดยไม่ต้องใช้ GraphQL server

การใช้งาน

// Client request: GET /api/products?fields=id,name,price

// ใน endpoint — apply() กรอง fields อัตโนมัติ
$products = Database::query("SELECT * FROM products");
Response::success(FieldSelector::apply($products));

// Whitelist ป้องกัน data leakage
FieldSelector::allow(['id', 'name', 'price']);
Response::success(FieldSelector::apply($products));

// Dot-notation สำหรับ nested fields
// GET /api/products?fields=id,category.name,tags.title

API Reference

Method การใช้งาน
FieldSelector::apply($rows) กรอง array ของ records
FieldSelector::applyOne($row) กรอง record เดียว
FieldSelector::allow($fields) Whitelist fields ที่อนุญาต
FieldSelector::requested() ดึง fields จาก ?fields= param
FieldSelector::respond($rows) apply + Response::success() ในคำสั่งเดียว

migrate ทยอยรันไฟล์ Migration ที่ยังไม่เคยถูกรันในฐานข้อมูล
migrate:rollback ย้อนสถานะ Database กลับไป 1 ชุด (Batch) ล่าสุด
migrate:status แสดงตารางรายชื่อไฟล์ Migration ทั้งหมด พร้อมบอกสถานะ (รันแล้ว/ยังไม่รัน)
mutate รันคำสั่ง Migration เริ่มต้น + เสก Table หายากเบื้องต้น (เช่น users) และ Seed ข้อมูลแอดมินคนแรก ลงไปให้ใช้งานทันที
seed นำข้อมูล Dummy (Seeders) ยัดใส่ Database ใช้สำหรับตอน Development
status ตรวจสอบการเชื่อมต่อกับ Database ว่ายังปกติอยู่หรือไม่ผ่าน PDO

3. คำสั่งกลุ่ม Caching System

จัดการ File-based Cache ในระบบแบบ Zero-dependency (เก็บลง `/storage/cache`)

คำสั่ง (Command) คำอธิบาย (Description)
cache:clear ล้างข้อมูลใน Cache ทั้งหมดให้ว่างเปล่า (Hard clear)
cache:stats แสดงสถิติ Cache (จำนวนไฟล์, Total Size, จำนวนไฟล์หมดอายุ, Tag usage)
cache:gc Garbage Collection ตระเวนลบไฟล์ Cache ที่หมดเวลา (Expired TTL) เพื่อคืนพื้นที่ Disk

4. คำสั่งกลุ่ม Genetic Marketplace & Modules

ค้นหาและดาวน์โหลดส่วนเสริมที่ช่วยให้คุณทำงานน้อยลง (เช่น 2FA, JWT Refresh, File Uploaders)

คำสั่ง (Command) คำอธิบาย (Description)
market:list แสดงรายชื่อ Packages ทั้งหมดที่มีในระบบ Registry
market:search <keyword> ค้นหา Package ด้วยชื่อหรือรายละเอียด
market:install <pkg> ติดตั้ง Package เข้ามายัง modules/ ของคุณทันที
เช่น: market:install og/notifications
modules:list เช็คดูว่า ณ ปัจจุบันมี Modules ไหนโหลดเข้ามาฝังในระบบแล้วบ้าง

5. คำสั่งระบบ (System)

คำสั่ง (Command) คำอธิบาย (Description)
serve เปิดหน้าต่าง Development Server บน PHP Router ผ่านพอร์ต 8080 (http://127.0.0.1:8080)
new <name> สร้างโครงสร้างเปล่าโปรเจกต์ OpenGenetics ตัวใหม่ พร้อม Composer Install
