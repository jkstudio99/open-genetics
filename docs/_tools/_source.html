OpenGenetics Documentation

Enterprise PHP Micro-Framework — JWT Auth, Genetic RBAC, i18n, Audit Trail, Dual-Frontend SDK.
ออกแบบมาเพื่อความเร็ว (<50ms), ความปลอดภัย (OWASP), และประสบการณ์นักพัฒนาที่ดี

< 50ms Response

No ORM, PDO Singleton, static cache

JWT + Bcrypt

HS256 auth, 12-round hashing

Genetic RBAC

Admin / HR / Employee guards

i18n Engine

Thai/English instant switching

Dual SDK

React Hook + Vanilla JS

Audit Trail

Non-blocking auto-logging

Philosophy: เล็ก เบา ทรงพลัง — Small, Light, Powerful. ทุก byte และ millisecond มีค่า

# การติดตั้ง

ความต้องการของระบบ

Requirement Version Notes
PHP >= 8.1 pdo, json, mbstring
MySQL >= 5.7 หรือ MariaDB 10.3+
Composer >= 2.0 Dependency manager
Apache XAMPP / any mod_rewrite enabled

เริ่มต้นด้วย Composer

5 ขั้นตอน — เหมือนติดตั้ง Laravel

1. ดาวน์โหลดหรือ Clone

composer create-project open-genetics/framework my-api
cd my-api

2. ติดตั้ง Dependencies

composer install

3. ตั้งค่า Environment

สร้างฐานข้อมูลใน phpMyAdmin แล้วแก้ไขไฟล์ .env

cp .env.example .env

4. Genetic Scaffolding

สร้างตาราง + Seed RBAC + Admin user อัตโนมัติ

php bin/genetics mutate

5. เริ่มสร้าง API

เขียน API ได้ทันที — ไม่ต้องประกาศ route

php bin/genetics serve

Default Admin: admin@opengenetics.io / password — เปลี่ยนรหัสผ่านทันทีหลัง deploy

# Configuration

ค่าทั้งหมดจัดการผ่านไฟล์ .env — class Env จะ load ครั้งเดียวแล้ว cache ใน static array

# .env
DB_HOST=localhost
DB_PORT=3306
DB_NAME=open-genetics-db
DB_USER=root
DB_PASS=

JWT_SECRET=your-super-secret-key-min-32-chars
JWT_EXPIRY=86400

APP_ENV=development
APP_DEBUG=true
APP_URL=http://localhost/open-genetics/public

CORS_ORIGIN=*
DEFAULT_LOCALE=th
Key Default Description
JWT_SECRET — ต้องยาวอย่างน้อย 32 ตัวอักษร
JWT_EXPIRY 86400 Token TTL (วินาที) = 24 ชั่วโมง
APP_DEBUG true ตั้งเป็น false ใน production
CORS_ORIGIN * ระบุ domain จริงใน production
DEFAULT_LOCALE th ภาษาเริ่มต้น: th หรือ en

# Architecture

OpenGenetics ใช้ File-based Routing — URL path ตรงกับ file path ใน api/ โดยตรง ไม่มี route registry

HTTP Method + URL File Class Method
POST /api/auth/login api/auth/login.php post(array $body)
GET /api/users api/users.php get(array $query)
PUT /api/users api/users.php put(array $body)
DELETE /api/users api/users.php delete(array $body)

Endpoint Template

use OpenGenetics\Auth\Guard;
use OpenGenetics\Core\{Database, Response};

class MyEndpoint {
    public static function get(array $query): void
    {
        Guard::requireAuth();
        $data = Database::query("SELECT * FROM my_table");
        Response::success($data);
    }

    public static function post(array $body): void
    {
        Guard::requireRole(Guard::ADMIN);
        Response::success(null, 'Created', 201);
    }
}

Scaffolding

$ php bin/genetics make:endpoint reports/monthly

  ✓ Created endpoint: api/reports/monthly.php
  Class: ReportsMonthly
  Route: GET|POST /api/reports/monthly

# Request Lifecycle

HTTP Request
    ↓

    ↓

    ↓
Router::dispatch()  →  resolve api/{path}.php
    ↓
Guard::requireAuth() / requireRole()   (optional)
    ↓
I18n::init()                           (optional, lazy)
    ↓
Class::method(params)
    ↓
AuditLog::log()                        (optional, non-blocking)
    ↓

    ↓

# Database

PDO Singleton — EMULATE_PREPARES=false (real prepared statements) ป้องกัน SQL Injection โดยค่าเริ่มต้น

query() — Fetch All

$users = Database::query(
    "SELECT * FROM users WHERE role_id = :role AND is_active = :a",
    ['role' => 2, 'a' => 1]
);

queryOne() — Fetch Single Row

$user = Database::queryOne(
    "SELECT * FROM users WHERE id = :id",
    ['id' => 1]
);

execute() — Insert / Update / Delete

Database::execute(
    "UPDATE users SET is_active = :a WHERE id = :id",
    ['a' => 0, 'id' => 5]
);

transaction()

Database::transaction(function (PDO $pdo) {
    Database::execute("UPDATE accounts SET balance = balance - 100 WHERE id = 1");
    Database::execute("UPDATE accounts SET balance = balance + 100 WHERE id = 2");
});

LIMIT/OFFSET: ใช้ intval() guard เพราะ MySQL ไม่รองรับ named params ใน LIMIT เมื่อ EMULATE_PREPARES=false

# Response

Response::success($data, 'OK', 200);
// {"success":true,"message":"OK","data":{...}}

Response::error('Validation failed', 422, ['email' => 'Required']);
// {"success":false,"message":"Validation failed","errors":{...}}

Response::paginated($rows, total: 100, page: 1, perPage: 20);
// {"success":true,"data":[...],"meta":{"total":100,"page":1,"per_page":20,"last_page":5}}
Method Status Description
success($data, $msg, $code) 200 ส่งข้อมูลสำเร็จ
error($msg, $code, $errors) 400–500 ส่ง error พร้อม details
paginated($rows, $total, $page, $perPage) 200 ส่งข้อมูลแบบ pagination
noContent() 204 สำเร็จ ไม่มีข้อมูลส่งกลับ

# Guard / RBAC

Guard ตรวจสอบ JWT token และ role ก่อนเข้า endpoint — throw RuntimeException พร้อม HTTP status อัตโนมัติ 

use OpenGenetics\Auth\Guard;

// ต้อง login เท่านั้น (role ใดก็ได้)
Guard::requireAuth();

// ต้องเป็น Admin เท่านั้น
Guard::requireRole(Guard::ADMIN);   // role_id = 1

// ต้องเป็น HR หรือสูงกว่า
Guard::requireRole(Guard::HR);      // role_id = 2

// ดึงข้อมูล user ปัจจุบัน
$user = Guard::user();
// ['id'=>1, 'email'=>'...', 'role_id'=>1, 'role_name'=>'admin']

// ตรวจสอบ role โดยไม่ throw
if (Guard::hasRole(Guard::ADMIN)) {
    // admin-only logic
}
Constant role_id Description
Guard::ADMIN 1 ผู้ดูแลระบบ — สิทธิ์สูงสุด
Guard::HR 2 HR — จัดการพนักงาน
Guard::EMPLOYEE 3 พนักงานทั่วไป

# Internationalization (i18n)

สลับภาษาทันทีผ่าน X-Locale header หรือ ?lang= param

I18n::t('auth.login');                       // "เข้าสู่ระบบ"
I18n::t('welcome', ['name' => 'John']);      // "Hello, John"
I18n::getLocale();                            // "th"
I18n::all();                                   // Full dictionary array

Lazy loading: i18n จะ initialize เฉพาะเมื่อมี X-Locale header หรือ ?lang= param เท่านั้น — ประหยัด ~0.5ms ต่อ request

Locale Files

// locales/th.json
{
  "auth.login": "เข้าสู่ระบบ",
  "auth.logout": "ออกจากระบบ",
  "welcome": "สวัสดี, {name}",
  "error.not_found": "ไม่พบข้อมูล"
}

# Audit Trail

AuditLog::log(AuditLog::CREATE, 'users', ['email' => 'new@example.com']);
AuditLog::log(AuditLog::LOGIN, 'auth');
AuditLog::log(AuditLog::DELETE, 'users', ['id' => 5]);

$logs  = AuditLog::query(userId: 1, action: 'CREATE', limit: 50);
$total = AuditLog::count(userId: 1);
Action Constant Value Use case
AuditLog::CREATE 'CREATE' สร้างข้อมูลใหม่
AuditLog::UPDATE 'UPDATE' แก้ไขข้อมูล
AuditLog::DELETE 'DELETE' ลบข้อมูล
AuditLog::LOGIN 'LOGIN' เข้าสู่ระบบ
AuditLog::LOGOUT 'LOGOUT' ออกจากระบบ

# Genetic SDK

Frontend SDK พร้อมใช้งานทั้ง React Hook และ Vanilla JS — ครอบคลุม auth, RBAC, i18n และ API calls

React Hook

import { useGenetics } from '../sdk/react/useGenetics';

function App() {
  const { user, login, logout, api, loading } = useGenetics();

  const handleLogin = async () => {
    await login('admin@opengenetics.io', 'password');
  };

  const fetchUsers = async () => {
    const data = await api.get('/api/users');
    console.log(data);
  };
}

Vanilla JS

import GeneticsSDK from '../sdk/vanilla/genetics.js';

const sdk = new GeneticsSDK({ baseUrl: 'http://localhost/open-genetics/public' });

// Login
const { token, user } = await sdk.login('email', 'password');

// Authenticated request
const users = await sdk.get('/api/users');

// Check role
sdk.hasRole('admin'); // true/false

// i18n
sdk.setLocale('en');
sdk.t('auth.login'); // "Login"

# CLI — Genetics Tool

เครื่องมือ command-line สำหรับ scaffold, migrate และ manage โปรเจกต์

php bin/genetics mutate            # สร้างตาราง + seed RBAC + admin user
php bin/genetics serve             # Start dev server http://127.0.0.1:8080
php bin/genetics make:endpoint     # Scaffold API endpoint file
php bin/genetics make:middleware   # Scaffold middleware
php bin/genetics docs:generate     # Generate OpenAPI JSON from source code
php bin/genetics new <name>        # Create new project
php bin/genetics help              # Show all commands

docs:generate

$ php bin/genetics docs:generate

  ✓ OpenAPI spec generated successfully: public/openapi.json

make:endpoint

$ php bin/genetics make:endpoint products/list

  ✓ Created: api/products/list.php
  Class: ProductsList
  Route: GET|POST /api/products/list
Command Description
mutate Run migrations + seed default RBAC data
serve PHP built-in dev server on port 8080
make:endpoint <path> Scaffold new API endpoint with class template
make:middleware <name> Scaffold middleware class
docs:generate Automatically generate OpenAPI definitions (openapi.json)
new <name> Create new OpenGenetics project

# Deployment

Apache .htaccess

RewriteEngine On
RewriteBase /open-genetics/public/

RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^(.*)$ index.php [QSA,L]

Production Checklist

# Item Action
1 APP_DEBUG=false ซ่อน error details จาก client
2 Strong JWT_SECRET openssl rand -hex 32
3 CORS_ORIGIN ระบุ domain จริงเท่านั้น
4 mod_rewrite Enable ใน Apache config
5 php bin/genetics mutate Verify tables ก่อน go-live
6 .env ใน .gitignore ห้าม commit secrets เด็ดขาด

XAMPP Production: ตั้งค่า APP_URL ให้ตรงกับ domain จริง และเปิด mod_headers สำหรับ CORS headers
