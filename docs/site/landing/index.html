Enterprise PHP Micro-Framework

PHP Micro-Framework ที่มาพร้อม JWT Auth, RBAC, Query Builder และ Genetic SDK — ทุกอย่างพร้อมตั้งแต่คำสั่งแรก ไม่ต้องเขียน boilerplate อีกต่อไป

เริ่มต้นใช้งาน
composer create-project open-genetics/framework my-api

ทำไมนักพัฒนาถึงเลือก OpenGenetics

คุณสมบัติ

ทุกอย่างที่ต้องการ อยู่ในกล่องเดียว

ไม่ต้องติดตั้ง package เพิ่ม — Auth, RBAC, Query Builder, Middleware, Cache, Testing พร้อมทันที

รวดเร็ว < 50ms

Micro-framework ที่เบาที่สุด — ไม่มี ORM overhead, PDO Singleton และ File-based Routing ตอบสนองใต้ 50ms

JWT Authentication

ระบบยืนยันตัวตน HS256 ในตัว พร้อม Bcrypt 12 รอบ รองรับ firebase/php-jwt และ native PHP fallback

RBAC 3 ระดับ

Guard ควบคุมสิทธิ์ Admin, Manager, Employee ในระดับ DNA ของทุก API request — เพียงบรรทัดเดียว

i18n Engine

สลับภาษา Thai/English ทันทีผ่าน X-Locale header หรือ ?lang= param — Lazy load ประหยัด ~0.5ms

Dual SDK

React Hook + Vanilla JS SDK พร้อมใช้งาน — ครอบคลุม auth, RBAC, i18n และ API calls ในไฟล์เดียว

Audit Trail

บันทึก log ทุก action อัตโนมัติแบบ Non-blocking — ติดตาม CREATE, UPDATE, DELETE, LOGIN ได้ทันที

Query Builder

NEW

Fluent Query Builder บน PDO — DB::table('users')->where('active',1)->paginate(20) พร้อม prepared statements ทุกคำสั่ง

Testing Framework

NEW

HTTP test client พร้อม actingAs(), seed() และ fluent assertions — ทดสอบ API ได้ทันทีโดยไม่ต้อง mock

Middleware Pipeline

Chain of Responsibility pattern — CORS, Auth, RateLimit พร้อม #[Middleware] attribute

Caching Layer

File-based cache พร้อม TTL, namespace isolation และ remember() pattern — ไม่ต้องติดตั้ง Redis

Admin Generator

สร้าง CRUD admin API อัตโนมัติจาก DB schema — php genetics make:admin users

Marketplace

ติดตั้ง community packages ผ่าน CLI — JWT Refresh, 2FA, File Upload, Notifications พร้อมใช้ทันที

ตัวอย่างโค้ด

โค้ดที่อ่านง่าย ผลลัพธ์ที่ทรงพลัง

สร้างไฟล์ → ได้ endpoint ทันที — ไม่มี routing config, ไม่มี boilerplate 

// api/auth/login.php — POST /api/auth/login
use OpenGenetics\Auth\AuthService;
use OpenGenetics\Core\Response;

$body = json_decode(file_get_contents('php://input'), true);

try {
    $result = AuthService::login($body['email'], $body['password']);
    Response::success($result, 'Login successful');
} catch (\RuntimeException $e) {
    Response::error($e->getMessage(), $e->getCode());
}
// api/users.php — GET /api/users (Admin only)
use OpenGenetics\Auth\Guard;
use OpenGenetics\Core\{Database, Response};

Guard::requireRole(Guard::ADMIN); // 401/403 อัตโนมัติ

$users = Database::query(
    "SELECT id, email, role_name FROM users WHERE is_active = 1"
);

Response::success($users);
// Query Builder — fluent API, prepared statements ทุกคำสั่ง
use OpenGenetics\Core\{DB, Response};
use OpenGenetics\Auth\Guard;

// GET /api/users — paginated, filtered, sorted
Guard::requireAuth();
$result = DB::table('users')
    ->select(['id', 'email', 'role_name', 'created_at'])
    ->where('is_active', 1)
    ->where('tenant_id', Guard::user()['tenant_id'])
    ->orderBy('created_at', 'DESC')
    ->paginate(20); // page từ ?page= อัตโนมัติ

Response::paginatedFrom($result);
// → {"success":true,"data":[...],"meta":{"total":...,"last_page":...}}

เริ่มต้นใช้งาน

จาก Terminal สู่ API พร้อมใช้

4 คำสั่ง — API ที่มี Auth, RBAC และ Database พร้อมทดสอบ

1

สร้างโปรเจกต์

composer create-project open-genetics/framework my-api

แล้ว cd my-api

2

ติดตั้ง Dependencies

composer install
3

ตั้งค่า Environment

cp .env.example .env

สร้างฐานข้อมูลใน phpMyAdmin แล้วแก้ไข DB_HOST, DB_NAME, DB_PASS

4

Genetic Scaffolding

php genetics mutate

สร้างตาราง + Seed RBAC + Admin user อัตโนมัติ — Default: admin@opengenetics.io / password

5

เริ่มสร้าง API

php genetics serve

เปิด dev server ที่ http://127.0.0.1:8080 — สร้างไฟล์ api/your-endpoint.php ได้ทันที
